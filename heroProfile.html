But How Do We Find the Hero ID ?!

This is where things begin to get a little complicated for the D3 Community API, but its nothing impossible.

The hero id is the number we just accessed in the Career Profile call.

player1.heroes[i].id

Right now, we will just copy and paste each of mukrumbus-1394's character's id's into the field to get the JSON objects

At the end of this how-To, we are going to nest these api calls together to pick a character from any player.

This request URL seems nearly identical to the request URL for the Career Profile Call. In fact, it only has one difference. Between the username and the locale we have "/hero/88905126". This is just the hero id number we pasted in above. We can replace this string with '/hero/' + a variable for the hero id, var heroID.

Placing this addition into our last request URL we made for the career profile yields this req.open:

req.open('GET', 'https://us.api.battle.net/d3/profile/' + Username1 + '/hero/' + heroID+ '?locale=en_US&apikey=' + apiKey, true);

Adding the hero id is the only difference in this request.

Now we add the event listener, when the listener receives information, we check the status. If req.status is between 200 and 400, we have success. We can now parse the responseText and use the data. We will call the parsed object hero.

var hero = JSON.parse(req.responseText);

Below are the JSON Objects returned from using the call for both of mukrumbus-1394's characters:

Some interesting information we can find in these JSON objects are:

The hero name is stored in the name property.

hero.name

The hero level is stored in the level property.

hero.level

The number of elite enemies the hero has killed is stored in an object named kills. This object has only one property, which is named elites and signifies the quantity of elites killed

hero.kills.elites

The hero's statistics are all stored in the property 'stats'. 'stats' is an object that holds each specific statistic of the hero. One either needs to know which stat that are looking for or loop through all of them. For example, intelligence is:

hero.stats.intelligence

There is one last property that is important to cover in the hero profile object. This property is items.

items is an object that contains objects which signify individual pieces of equipment. What is interesting and unique about this object is that properties do not exist if the character does not have items equipped. For instance scroll through sandWitch's and YMIR's JSON objects. sandWitch has an items.waist, items.rightFinger, and items.leftFinger property, where YMIR does not.

This is an important quality of the items object in terms of scripting. In order to not create errors in one's code, logic statements must be used. If one wants to display the hero's head equipment in a particular location, he must first make sure the character has a head property via the hasOwnProperty function.

if(items.hasOwnProperty('head')){ ... };

There is one final piece of information needed to use the items object to its full ability, and that is how to find the images associated with the objects. This is no way to read through the D3 documentation to find out how to access the images, but snooping through the pages of Diablo 3 Community will let you discover that all item images used in the D3 api are found at https://blzmedia-a.akamaihd.net/d3/icons/items/.

Entering that into your browser will declare an error, but this URL will be used for the beginning string. There are too options, either using the small or the large directory in this url. We will use the large. Let's make this URL a variable called imageURL.

var imageURL = 'https://blzmedia-a.akamaihd.net/d3/icons/items/large/';

Now That we have the directory, we can specify which image we want from the hero JSON object. Notice how each object in hero.items has an icon property. This is the name of the image we are looking for. Append this name to imageURL

Finally, We know exactly where to find the image and the only thing left to do is specify the file type. These images are .PNG files, so append ".png" to the entire string as follows:

imageURL + hero.items[prop].icon + ".png";
